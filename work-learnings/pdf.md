# PDF Generation Learnings and Architecture Notes

## Overview

While developing the **Custom Quotation Management System**, one of the most technically challenging modules was **PDF generation**.

The requirement was not limited to exporting HTML as PDF. The final system needed to generate enterprise-grade quotations containing:

* Dynamic customer details
* Pricing information
* Multiple elevations
* Technical line drawings
* Company branding
* Watermarks
* Multi-page support
* Mixed page sizes (A4 + A3)
* Automatic file cleanup

This document summarizes all major concepts learned during the implementation.

---

# 1. Understanding Different PDF Libraries

Multiple PDF libraries were evaluated during development.

## Dompdf

Dompdf converts HTML and CSS into PDF.

### Advantages

* HTML/CSS based.
* Easy integration with existing templates.
* Excellent for invoices and quotations.
* Supports tables, styling, images, and page breaks.

### Limitations

* Limited support for advanced PDF manipulation.
* Difficult to merge multiple PDFs.
* Limited control over imported pages.
* Complex watermarking for mixed layouts.

### Use Case in Project

Used for generating:

* Quotation pages
* Customer information pages
* Pricing tables
* Company information sections

---

## FPDF

FPDF generates PDFs programmatically using PHP methods.

Example:

```php
$pdf->Cell();
$pdf->Text();
$pdf->Image();
```

### Advantages

* Lightweight.
* Full control over PDF positioning.
* Easy footer/header implementation.

### Limitations

* No native transparency support.
* No HTML rendering.
* Limited typography support.

---

## FPDI

FPDI extends FPDF/TCPDF.

Purpose:

```text
Import existing PDFs as templates.
```

### Features

* Import PDF pages.
* Merge multiple PDF documents.
* Combine PDFs of different page sizes.

Example:

```php
$pageCount = $pdf->setSourceFile($file);

$template = $pdf->importPage(1);

$pdf->useTemplate($template);
```

### Use Case

Used to:

* Import A4 quotation pages.
* Merge A4 and A3 documents.
* Build final quotation PDF.

---

## TCPDF

TCPDF is an advanced PDF generation library.

### Advantages

* Native transparency support.
* Rotation support.
* Unicode support.
* Better typography.
* Advanced drawing capabilities.

### Features Used

```php
$pdf->SetAlpha();
$pdf->Rotate();
$pdf->StartTransform();
$pdf->StopTransform();
```

### Use Case

Used for:

* Watermark generation.
* A3 elevation pages.
* Footer generation.
* Advanced PDF customization.

# 3. Dynamic HTML to PDF Conversion

Process:

```php
ob_start();

include 'invoice-template.php';

$html = ob_get_clean();

$dompdf->loadHtml($html);
```

### Concept

PHP output buffering captures HTML generated by templates.

Functions:

```php
ob_start()
ob_get_clean()
```

This allows dynamic data insertion into PDF templates.

# 4. Image Handling in PDFs

Several issues occurred while rendering elevation drawings.

## Common Problems

### Image exists in browser but not PDF

Reasons:

* Wrong filesystem path.
* Relative paths not supported.
* Missing permissions.
* Invalid MIME type.

---

## Verification Techniques

### Check file existence

```php
file_exists($path)
```

### Get absolute path

```php
realpath($path)
```

### Determine MIME type

```php
mime_content_type($path)
```

### Get image dimensions

```php
getimagesize($path)
```

# 5. Base64 Image Embedding

Images were converted to Base64 for Dompdf compatibility.

Example:

```php
$imageData = base64_encode(
    file_get_contents($imagePath)
);

$imageSrc =
'data:image/png;base64,' .
$imageData;
```

Benefits:

* Avoids path issues.
* Portable.
* Self-contained HTML.

# 6. Watermark Implementation

Business requirement:

```text
Enterprise watermark on all drawing pages.
```

Initially attempted using:

* FPDF rotation.
* Custom transparency classes.

Final solution:

```php
$pdf->SetAlpha(0.05);

$pdf->StartTransform();

$pdf->Rotate(45,180,140);

$pdf->Text(...);

$pdf->StopTransform();

$pdf->SetAlpha(1);
```

Concepts learned:

## Transformation Matrix

Transforms modify coordinate systems.

Examples:

```php
Rotate()
Scale()
Translate()
```

---

## Transparency

```php
SetAlpha()
```

Controls watermark opacity.

Example:

```php
0.05 = 5% visible
```

# 9. Supporting Mixed Page Sizes

Requirement:

```text
A4 → Quotation
A3 → Drawings
```

FPDI imports each page while preserving size.

Example:

```php
$size =
$pdf->getTemplateSize($tpl);

$pdf->AddPage(
    $size['orientation'],
    [$size['width'],$size['height']]
);
```

# 10. Temporary File Management

PDF merging required temporary files.

Example:

```php
file_put_contents(
    $a4Temp,
    $pdfData
);
```

### Problem

Repeated exports created unnecessary files.

Solution:

```php
unlink($a4Temp);
```

Old files cleanup:

```php
foreach (
    glob($tempDir.'*.pdf')
    as $file
) {

    if (
        filemtime($file)
        < time() - 600
    ) {

        unlink($file);
    }
}
```

# 12. Key Takeaways

* HTML to PDF is not always straightforward.
* File path management is critical.
* Unicode requires proper font support.
* Mixed page sizes require PDF merging.
* Temporary files must be cleaned automatically.
* Watermarks are easier using TCPDF.
* FPDI is essential for importing and merging PDFs.
* Debugging generated HTML significantly reduces troubleshooting time.
* Enterprise PDF generation often requires multiple libraries working together.
